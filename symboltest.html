How are basic math symbols read out in various screen readers?

The text below is taken from the Powerpoint Guidelines.

Setting MathType Equations

Keyboard characters that should be used only from within MathType to create a MathType equation:

  • Letters
  • Numbers
  • Minus sign
  • Plus sign
  • Equal sign
  • Comma
  • Exclamation point
  • Dollar
  • Percent
  • Less than
  • Greater than

These characters must be applied from the MathType toolbar:

  • Tilde
  • Similar to
  • Prime
  • Caret
  • Asterisk
  • Parentheses
  • Brackets
  • Angle brackets
  • Braces
  • Less than or equal to
  • Greater than or equal to
  • Multiplication dot (called the "dot operator", use the smallest of the MathType pull-down choices; using this symbol will also include space on either side of the operator automatically)

NVDA note: What's written out for NVDA (all) is omitting the "dot" spoken for the periods separating symbols.

Note on the Ohm symbol: It triggers a warning from the W3C: "Text run is not in Unicode Normalization Form C."

From StackOverflow: The reference Ω (Ω) triggers the message, because it's defined to be canonical equivalent to Omega (Ω). The Unicode Standard explicitly says that the latter is the preferred character and is also better covered in fonts.
